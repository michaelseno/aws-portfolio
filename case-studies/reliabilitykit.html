ReliabilityKit — MVP Case Study

ReliabilityKit is a local-first reliability execution layer that wraps pytest + Playwright runs and converts each run into structured reliability data for faster triage and trend analysis.

Problem

  • Failed E2E runs are often noisy and hard to compare historically.
  • Teams need deterministic chaos runs and repeatable diagnostics.
  • Per-run reports alone do not expose reliability drift over time.

System Design

  • CLI-first execution with Typer commands for run, inspect, report, trend, and dashboard.
  • Pytest plugin captures runtime records, artifacts, chaos events, and failure digests.
  • Local-first storage under `.reliabilitykit/` with S3 backend architecture staged.
  • HTML dashboard unifies latest-run triage and historical reliability views.

Reliability Signals

  • Run reliability score and test reliability score.
  • Flake rate, chaos sensitivity, and failure diversity.
  • Failure cluster aggregation using digest fingerprints.
  • Scheduled CI to maintain continuous signal freshness.

Status and Roadmap

  • Status: MVP complete and actively evolving.
  • Implemented: unit tests, e2e suites, golden checks, architecture guardrails, scheduled CI.
  • Next: production-grade S3 backend and deeper trend analytics.
